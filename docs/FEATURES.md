# Фичи

## 1. Авторизация

### Описание
Пользователь логинится через email/password и получает JWT токен.

### Поток
1. Ввод логина/пароля
2. Запрос в API
3. Сохранение токена в Keychain
4. Переход на главный экран

---

## 2. Генерация QR-кода

### Поведение
- Генерируется по кнопке
- Валиден 5 минут
- После истечения — инвалидируется
- При повторном открытии — генерируется новый

### Payload
(определяется backend, но должен быть абстрактным)

---

## 3. Блокировка приложения

### Методы
- PIN-код
- Face ID

### Поведение
- При запуске → экран блокировки
- При уходе в background → блокировка

---

## 4. Защита экрана

### Реализация
- Отслеживание screen capture notifications
- Блюр при background

---

## 5. Сессия пользователя

- Пользователь остаётся залогинен
- Токен хранится в Keychain
